# StarrOS Playbook Draft

Status: Draft v0
Owner: J7
Reviewers: San, Jigo, Ishii, Kohroku

## Purpose

Define how StarrOS is built end-to-end through a repo-first, kanban-driven workflow.

## Core rules

- This repository is the canonical planning and execution surface.
- No hidden planning artifacts are authoritative unless committed here.
- San orchestrates; wolves specialize; external coders implement only bounded leaf cards.
- High-risk changes require explicit human approval.
- Every completed card must produce artifacts, logs, and a review outcome.

## Required sections

- Executive intent
- System-of-truth map
- Wolf responsibilities
- Kanban states and transitions
- Spec workflow
- Review/sign-off protocol
- Artifact contract
- Escalation rules
- Anti-patterns
- Phase-based delivery sequence
