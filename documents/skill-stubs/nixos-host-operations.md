# `nixos-host-operations` — Skill Stub

**Purpose.** Safe operating procedure for San (agent on Mononoke NAS) when managing NixOS host, flakes, services, permissions, and rebuild workflows. Covers the load-bearing quirks that make Mononoke's NixOS host operations different from a stock Linux box: per-user Nix profile silent no-op, the four-layer `ExecStartPre=` bug chain in user drop-ins, `nixos-rebuild switch` collision patterns, kanban-blocked-but-live-state reconciliation, and the hermes-agent / wolfpack gateway activation matrix.

**When to load.** Any time San is about to: (1) run `nixos-rebuild` (build, switch, test, boot — any variant), (2) edit files under `/etc/nixos/nix/` or modify `/etc/nixos/flake.nix`, (3) create/modify/remove a systemd unit that the NixOS config owns, (4) debug a service stuck in `activating (auto-restart)` or `inactive (dead)` when it should be active, (5) touch `services.hermes-agent`, `services.wolfpack`, or any of the 10 `hermes-gateway-<wolf>.service` units, (6) read a user drop-in's failing `ExecStartPre=` journal entry (see `references/execstartpre-user-drop-in-quirks-2026-07-06.md`), (7) run any irreversible op against a kanban card.

**What it forbids.** Running `nixos-rebuild switch` without J7 same-turn sign-off. Skipping the dry-run-verify-nix-module-change-systemd-user-and-system procedure. Treating a "kanban says blocked" reading as authoritative when live state (gen X active, ports reassigned) proves the work was already done in a prior session. Re-implementing canonical Mononoke patterns (ssh hang workaround, manual systemd overrides, dual-source-of-truth Hermes deployment) without consulting the references first.

**Canonical home.** `/home/san/.hermes/skills/devops/nixos-host-operations/SKILL.md` (v3.14.0, 2026-07-06). Reference docs and helper scripts at `references/` and `scripts/` within that skill directory. The `scripts/bsm-secret-fetch-for-environmentfile.sh` helper is the verified-good template for user-drop-in `ExecStartPre=` BWS secret loading — unrelated to Mnemosyne restart (that uses the `mnemosyne_dashboard_start` Hermes tool; see E3-T17).
